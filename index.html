linux-who

显示目前登入系统的用户信息

-H or –heading：显示各栏位的标题信息列
-u：显示闲置时间，若该用户在前一分钟之内有进行任何动作，则标识“.”，如

-q or –count：只显示登入系统的帐号名称和总人数

可选的操作系统组件和第三方应用程序可能被安装在 /opt 目录下

yaml 语法

发表于 2016-09-02   |   分类于 其他

yaml是专门用来写配置文件的语言，非常简洁强大，远比 json 方便

基础语法规则

  • 大小写敏感
  • 使用缩进表示层级关系
  • 缩进时不允许使用 tab 键，只允许空格键
  • 缩进空格数目不重要，只要相同层级的元素左侧对齐即可
  • # 表示注释

yaml 支持的数据格式

  • 对象：一组键值对，冒号结构表示
  • 数组：一组连词线开头的行，构成一个数组，缩进可以构成子数组
  • 纯量：是最基本的、不可再分的值。字符串、布尔值、整数、浮点数、null、时间、日期

注意

  • null 用 ~ 表示
  • 时间采用 ISO8601格式，2001-12-14t21:59:43.10-05:00
  • 日期采用复合 ISO8601 格式的年、月、日表示
  • 允许使用 !! 强制转换数据类型

字符串是最常见，也是最复杂的一种数据类型，默认不用引号表示，若包含特殊字符，则需要引号

& 用来建立锚点，<< 表示合并到当前数据，* 用来引用锚点

Linux 学习笔记02 -- Linux 文件属性详细说明

发表于 2016-09-02   |   分类于 Linux

Linux 文件或目录的属性主要包括：文件或目录的节点、种类、权限模式、链接数量、所归属的用户和用户组、最近访问或修改的时间等内容。

inode

inode，索引节点。每个存储设备或存储设备的分区（存储设备拉硬盘、U盘等）被格式化为文
件系统后，应该有两部分：
    1. inode：用来存储数据的信息，如文件的大小、属主、读写权限等
    2. block：用来存储数据

链接文件

  • 用 ln 可以创建链接文件，语法：ln 源文件 目标文件
  • 在 Linux 文件系统中，inode 值相同的文件是硬链接文件
    • 硬链接不能为目录创建
    • 当一个其中一个硬链接文件的内容，互为硬链接文件的内容也会变化，若是删除时，其他不影响
  • -s 选项可以创建软链接，链接文件不能独立存在

C 学习笔记_网络与套接字

发表于 2016-09-02   |   分类于 C

协议，客户端与服务端的结构化对话
套接字不是传统意义上的数据流
他是双向的，他既可以用作输入也可以用作输出

BLAB 服务器连接网络四部曲

  • B 绑定端口，
  • L 监听
  • A 接受连接
  • B 开始通信

注意： 当某个端口被绑定了套接字，在接下来的30秒内，操作系统不允许任何程序再绑定它

从客户端读取数据

<读了几个字节> = recv(<描述符>, <缓冲区>, <要读取几个字节>, 0);

注意：

  • 字符串不以 \0 结尾
  • 当用户在 telent 输入文本时，字符串以 \r\n 结尾
  • recv() 将返回字符个数，如果发生错误就返回-1，如果客户端关闭了连接，就返回0
  • recv() 调用不一定能一次接收到所有字符

客户端：

  • 连接远程端口
  • 开始通信

创建域名套接字

getaddrinfo() 获取域名的地址，需要 netdb.h 文件，会在堆上创建一种名字资源的新数据结构
