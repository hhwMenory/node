进程与信号

发表于 2016-10-19   |   分类于 c

swoole_process

创建子进程

int swoole_process::__construct(
    mixed $function, bool $redirectStdinStdout = false, bool $createPipe = true
)

$function：子进程创建成功后要执行的函数
$redirectStdinStdout：是否重定向子进程的标准输入从管道读取和输出到管道
$createPipe：是否创建管道，启用 $redirect_stdin_stdout 后强制为 true

$process->pid：子进程的 PID
$process->pipe：管道的文件描述符

启动进程

// 执行fork系统调用启动进程，成功返回子进程的 PID，创建失败返回 false
[int | false] swoole_process->start()

if (swoole_process->start() === false) {
    // 使用 swoole_errno 和 swoole_strerror 得到错误码和错误信息
}

// 修改进程名称
bool swoole_process::name(string $newProcessName)
eg：$process->name("php server.php: worker")

守护进程化

// 使当前进程脱变为一个守护进程
bool swoole_process::daemon(bool $nochdir = false, bool $noclose = false)

$nochdir：true 表示不修改当前目录。false 表示将当前目录切换到 “/”
$noclose：表示是否将标准输入和输出重定向到 /dev/null

管道通信

// 向管道内写入数据
int swoole_process->write(string $data)

    1. $data 的长度在 Linux 系统下最大不超过 8K，MacOS/FreeBSD 下最大不超过 2K
    2. 在子进程内调用write，主进程会收到数据
    3. 在主进程内调用write，子进程会收到数据

// 从管道中读取数据
int swoole_process->read(int $bufferSize = 8192)
