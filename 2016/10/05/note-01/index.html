linux-who

显示目前登入系统的用户信息

-H or –heading：显示各栏位的标题信息列
-u：显示闲置时间，若该用户在前一分钟之内有进行任何动作，则标识“.”，如
果该用户已超过24小时没有任何动作，则标志“old”
-m：同 who am i
-q or –count：只显示登入系统的帐号名称和总人数

/bin：二进制文件目录，用于存放启动系统时用到的程序
/usr/bin：用户二进制文件目录，用于存放用户使用的标准程序
/usr/local/bin：本地二进制文件目录，用于存放软件安装的程序
系统用户管理员登录后使用的 PATH 变量可能还包含存放系统管理程序的目录 /sbin 和 /usr/sbin
可选的操作系统组件和第三方应用程序可能被安装在 /opt 目录下

shell 语法

变量

  • 在 shell 使用变量之前通常并不需要声明
  • 默认情况下，所有的变量都被看作字符串并以字符串来存储
  • 在变量名之前加上一个 $ 符号来访问它的内容
  • 字符串里包含空格就必须要用引号括起来
  • 等号两边不能有空格
  • read 命令可以将用户的输入赋值给一个变量，回车键结束命令

注意：

  1. 环境变量

    1. $HOME：当前用户的家目录
    2. $PATH：以冒号分隔的用来搜索命令的目录列表
    3. $PS1：命令提示符，通常是 $ 字符，但在 bash 中，可以使用一些更复杂的值，[\u@\h \W]\$，即[用户名@机器名 当前目录名]$提示符
    4. $PS2：二级提示符，用来提示后续输出，通常是 > 字符
    5. $IFS：输入域分隔符，当 shell 读取输入时，用来分隔单词的一组字符，通常是空格、制表、换行
    6. $0：shell 脚本的名称
    7. $#：传递给脚本的参数个数
    8. $$：shell 脚本的进程号，脚本程序通常会用它生成一个唯一的临时文件。如 /tmp/tmpfile_$$

  2. 参数变量
    如果脚本程序在调用时带有参数，一些额外的变量就会被创建，即使没有传递任何参数，环境变量 $# 也依然存在，只是值为0

    1. $1, $2, …：脚本程序的参数
    2. $*：在一个变量中列出所有的参数，各个参数之间用 $IFS 中的第一个字符分隔
    3. $@：$* 变体，不使用 $IFS

条件

test 或 [ ：对条件进行测试判断，可以使用的条件类型大概分为三类：

  • 字符串比较
    • = 两个字符串相同则结果为真
    • != 两个字符串不同则结果为真
    • -n string 字符串不为空则结果为真
    • -z string 字符串为 null（一个空串）则结果为真
  • 算术比较
    • -eq -ne -gt -ge -lt -le !
  • 与文件相关的条件测试
    • -d [-e|-f]
    • -r（可读） -w（可写） -x（可执行）
    • -s 文件大小不为0则结果为真
    • -u 文件的 set-user-id 位被设置则结果为真
    • -g 文件的 set-group-id 位被设置则结果为真

控制结构

命令

1. break 命令：跳出循环
2. : 命令：是一个空命令，相当于 true 的别名
3. continue 命令：跳到下次循环
4. echo 命令：建议使用 printf 命令，用法类 c 
5. trap 命令：用于指定在接收信号后将要采取的行动

命令执行

1. 个人比较喜欢 $(command) 的方式来实现，也可以使用 `command`，返回结构为命令的输出
2. $((...)) 可以求值表达式
3. 参数扩展：
    ${param:-default} 若 param 为空，则返回 default，而 param 并未变化
    ${param:=value}   若 param 为空，则将 param 赋值为 value
    ${param:?value}   若 param 为空，则输出 param:value 并异常终止脚本
    ${param:+value}   若 param 存在且不为空，则返回 value
    ${#param} param 的长度
    ${param%word}  从 param 尾部删除最短匹配
    ${param%%word} 从 param 尾部删除最长匹配
    ${param#word}  从 param 头部删除最短匹配
    ${param##word} 从 param 头部删除最长匹配

调试脚本程序

  • -n | set -n：只检查语法，不执行命令
  • -v | set -v：在执行命令之前回显它们
  • -x | set -x：在执行命令之后回显它们
  • -u | set -u：若使用了未定义的变量，就给出错误信息

文件操作

Linux 文件结构

文件，除了本身包含的内容外，它还会有一个名字和一些属性，即“管理信息”。而在 Linux 中可以说一切都是文件，这意味着程序通常可以像使用文件那样使用磁盘文件、串行口、打印机和其他设备。

目录：是用来保存其他文件的节点号和名字的文件。删除一个文件，即删除了该文件对应的
目录项，同时该文件的链接数减1，若链接数为0表示该节点指向的数据不再使用。

文件和设备：
    /dev/console：系统控制台，错误信息和诊断信息通常会被发送到这个设备
    /dev/tty：
    /dev/null：所有写向这个设备的输出都会丢弃
