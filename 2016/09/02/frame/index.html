个人开发框架

发表于 2016-09-02   |   分类于 PHP

前言

作为一个编程开发的新手，我很喜欢去尝试和学习一些新的技术和知识，来提升自己的个人能力，
个人也偏好学习一些底层相关的知识，虽然这几年来的学习让我有了很大的提升，但是自己感觉学
习的东西很多，也很杂。有些东西学习后，因为公司的开发条件限制只能是了解过，并没有真正的
去实践它，于是便有了自己编写个人开发框架的想法，编写框架可以很好的去实践自己的一些想法
，也能去思考一些架构的问题，更可以糅合自己掌握的相关技能。

概述

menory-framework 的运行环境是基于 PHP7、Nginx 和 Swoole 扩展的，只要是使用 Swoole 做应
用服务，Nginx 做代理转发。menory-framework 简单可以划分为框架、服务和应用三个部分，服务
是基于 Swoole 扩展而给应用相关的网络服务，而应用可以有多种，例如以 MVC 模式为主的 Web
应用，最后的框架则是服务和应用的糅合剂，给他们提供一个可运行环境。

目录结构

框架目录

/app 一些内置的框架管理应用目录
/bin 管理脚本目录
/etc 应用服务的配置文件目录
/factory ###
/var 存放应用服务运行时要改变的数据 pid, log
/vendor 框架使用 composer 来管理依赖关系，效果同 composer 的设定

注意：

  • 框架的核心代码也以第三方扩展的方式存放在 /vendor/menory
  • /var/pid/{appName} 存放应用服务运行时的进程 pid，如 master 和 manager 进程的pid

应用目录（MVC 模式）

/Controller 控制器文件目录
/Configs 应用级配置文件目录
/Hooks 钩子文件目录
/View 视图文件目录
/Libs 应用库目录

框架使用

框架的使用很简单，主要可以分为下面几个步骤：

  • 使用 composer 安装框架：composer create-project menory/menory-framework frame “1.0.0”
  • 运行配置文件创建脚本生成相关配置：php bin/createConf.php [应用名称]
  • 进行相关的 Nginx 配置
  • 运行应用启动脚本：php bin/app.php start [应用名称] –debug（调试模式） –daemon（后台运行）

定时器

定时器，周期性的执行某项任务。在实际的开发中，定时任务是非常常用的，它可以把一些复杂的
业务逻辑解耦，例如商品的开奖、分佣、审核等等，也可以用来实现倒计时开奖，消息通知失败的
重试机制等等。而框架中根据 swoole 的特点，也考虑到怕一些费时定时任务影响其他定时任务的
执行，所以采用了 swoole 的进程管理模块和毫秒定时器来实现，让每一个定时任务都是一个独立
的进程。
